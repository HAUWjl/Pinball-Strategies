# Pinball-Strategies
面对中国目前流行的12口弹珠机，指定最佳出卡策略。在同一机器不断游玩时，投珠子策略可根据每次结果进行调整，玩家每次将拉杆拉到底即可。

```
main.py               # 交互式命令行工具

```

## 弹珠机规则

| 倍数 | 亮灯格数 | 中奖概率 | 珠子期望 ROI |
|------|---------|---------|------------|
| 2x   | 4       | 1/3     | 0.667      |
| 4x   | 3       | 1/4     | 1.000      |
| 6x   | 2       | 1/6     | 1.000      |
| 8x   | 1       | 1/12    | 0.667      |
| 10x  | 1       | 1/12    | 0.833      |

- 每局最少投入 **5** 颗小珠才能开始游戏，最高可追投至 **99** 颗。  
- 中奖时返回 `倍数 × 投珠数` 颗珠子，并获得 `min(floor(倍数 × 投珠数 / T), J)` 张积分卡。  
  其中 **T** 为积分卡分母（每 T 个返珠获得 1 张积分卡，典型值 20~50），**J** 为单次中奖积分卡上限。

---

## 参数说明

| 参数 | 说明 |
|------|------|
| **T** | 积分卡分母：每 T 个返珠（投珠数 × 倍数）获得 1 张积分卡。典型值 20~50，由机器决定 |
| **J** | 单次中奖最多可获积分卡数（由机器决定） |
| **优先目标** | `cards` = 积分卡优先；`marbles` = 珠子收益优先 |
| **最低投珠** | 每局至少投 **5** 颗小珠（机器硬限制，与 T 无关） |

---

## 策略核心

### 珠子收益优先（`marbles`）
- 仅当 `倍数 × 中奖概率 > 1` 时才追投到最大（99颗）；否则只投最低的 5 颗。  
- 在均匀落点假设下，4x 和 6x 为收支平衡点，其余倍数均为负期望。

### 积分卡优先（`cards`）
- 每次中奖积分卡数 = `floor(倍数 × 投珠数 / T)`，上限 J。  
- 每珠积分产出 = `P(中奖) × min(floor(倍数 × N / T), J) / N`。  
- 当 N 足够大使得 `倍数 × N / T ≥ J` 时，增加 N 反而降低每珠产出，因此最优投珠数为：  
  `N* = max(5, min(99, ⌈T × J / 倍数⌉))`

### 落点自适应
每局结束后记录弹珠落入的槽位，程序会逐渐学习该机器的物理落点偏好，并据此调整中奖概率估计，使后续建议更加精准。

## 快速上手

```bash
python main.py
```

按提示输入机器的 T、J 值和优先目标，然后循环进行每局操作：

1. 输入本局倍数（2/4/6/8/10）
2. 输入亮灯槽位编号
3. 程序输出**建议投珠数**及期望收益
4. 输入本局结果并记录落点槽位（可选）
5. 随着记录增多，概率估计越来越准确

## 编程接口

```python
from pinball_strategy import PinballStrategy

# 初始化（T=20 为积分卡分母，J=10 张上限，积分卡优先）
s = PinballStrategy(T=20, J=10, priority="cards")

# 某局：4 倍数，亮灯槽位为 2、7、11（0-indexed）
rec = s.recommend(multiplier=4, lit_slots=[1, 6, 10])
print(rec["optimal_bet"])          # 建议投珠数：⌈T×J/倍数⌉ = ⌈20×10/4⌉ = 50
print(rec["expected_score_cards"]) # 期望积分卡数

# 记录本局落点（弹珠落入槽位 7，0-indexed = 6）
s.record_landing(6)

# 查看当前落点概率分布
print(s.get_landing_probs())

# 查看基准期望值表
rows = PinballStrategy.expected_value_table(T=20, J=10)
```
