# 12口弹珠机最佳出卡策略

针对中国流行的 **12口弹珠机**（弹珠落入亮灯格中奖），基于贝叶斯推断实时学习落点概率，自动给出每局最优投珠建议，最大化积分卡产出或珠子收益。

---

## 机器规则简述

1. **投珠开局** — 投入 5~99 颗小弹珠，按按钮确定**倍率**（2×/4×/6×/8×/10×）。
2. **亮灯分布** — 倍率决定亮灯数：2× 亮 4 格、4× 亮 3 格、6× 亮 2 格、8×/10× 亮 1 格。灯格在 12 个落点中随机选取。
3. **可追加投珠** — 看到倍率和亮灯后，发射前可继续加珠（上限 99）。
4. **发射判定** — 弹珠自然滚落；落入亮灯格 → 中奖返珠 = 倍率 × 投珠数；未亮灯 → 珠子全损。
5. **积分卡奖励** — 中奖时额外获得积分卡：`cards = min(floor(倍率 × 投珠 / T), J)`。其中 **T** 为每多少返珠获得一张积分卡，**J** 为单次出积分卡上限。不同机器的 T 和 J 不同。

| 倍率 | 出现概率 | 亮灯数 | 中奖率（均匀分布） | 珠子 ROI |
|------|---------|--------|------------------|---------|
| 2×   | 42.0%   | 4      | 33.3%            | 0.667   |
| 4×   | 28.8%   | 3      | 25.0%            | 1.000   |
| 6×   | 12.7%   | 2      | 16.7%            | 1.000   |
| 8×   | 10.8%   | 1      | 8.3%             | 0.667   |
| 10×  | 5.7%    | 1      | 8.3%             | 0.833   |

## 核心策略：V2 自适应投注

### 原始策略的问题

原始积分卡策略始终投注 `⌈T × J / mult⌉` 颗，在概率未知的早期阶段也一样。这导致在**负期望值**局面下大量浪费珠子。

### V2 关键洞察

| 期望值 | 投注策略 | 原因 |
|--------|---------|------|
| **负 EV**（mult × p_win ≤ 1）| 投最小步进量 `n_floor = max(5, ⌈T/mult⌉)` | 小注的**每珠积分效率**相同甚至更高（避免 floor() 取整浪费），同时提供廉价的概率探索数据 |
| **正 EV**（mult × p_win > 1）| 按**信心系数**从 n_floor 逐步加码至 99 | 正 EV 时投大注不仅赚更多积分卡，赢回的珠子还能延长游戏产生**滚雪球效应** |

信心系数定义：

```
confidence = 已打局数 / (已打局数 + confidence_threshold)
```

`confidence_threshold` 是唯一的超参数，控制加码速度。**推荐值为 5**。

## 技术原理

### 贝叶斯落点概率估计

使用 **Dirichlet 先验 + 多项式似然** 估计 12 个槽位的落点概率：

```
P(slot_i) = (α + count_i) / (prior_weight + total_plays)
```

其中 `α = prior_weight / 12`。先验强度为 24 时，等价于预先假设每个槽位观测到 2 次落点（均匀分布）。随着游戏进行，实际观测逐渐压过先验，概率估计趋近真实分布。

### 期望值计算

- **中奖概率**：`p_win = Σ P(slot_i)` for i in lit_slots
- **珠子 ROI**：`EV = mult × p_win`（> 1 为正收益）
- **积分卡期望**：`E[cards] = p_win × min(floor(mult × bet / T), J)`

### 最小步进量

对于负 EV 局面，`n_floor = max(5, ⌈T / mult⌉)` 是能赢得 1 张积分卡的最小投注。此时每颗珠子的积分效率：

```
efficiency = floor(mult × n_floor / T) / n_floor
```

这个值在 `n_floor` 处最大或与更高投注相同（因为 `floor()` 取整让大注浪费珠子但不多产卡）。

### 自适应加码逻辑

```
if mult × p_win ≤ 1:       → 投 n_floor（负 EV，低成本探索）
if mult × p_win > 1:       → 投 n_floor + confidence × (99 - n_floor)（正 EV，按信心放大）
```
